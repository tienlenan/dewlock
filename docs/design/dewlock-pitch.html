The problem & the model

Acting on Sui DeFi today means trusting a black box.

Liquidity is scattered across Cetus, DeepBook, SuiNS. The natural-language layer everyone wants usually means handing an agent your keys, or signing bytes you never read. Dewlock inverts it: the agent builds, the Guardian verifies, and you sign the exact artifact — never a key on the server.

New · one-signature chaining

Any actions. One signature. All-or-nothing.

“Swap 2 SUI to USDC then lend it on NAVI” compiles into a single PTB you sign once — the swap output feeds the deposit inside the transaction. A Move abort in either leg reverts both. No half-done state, no second prompt. Any allowlisted combo composes the same way — pay several friends at once, swap→lend, stake→send — up to 8 legs, one signature.

How it composes

Composable swap

The Cetus aggregator’s routerSwap returns the output coin as a PTB argument — fed straight into the NAVI deposit. No wallet round-trip between legs.

structural proof: swap-out = lend-in
Still fail-closed

One Guardian recipe

checkCompositeRecipe re-verifies the whole PTB: closed recipe, exact target multiset, coin-type linkage, delta/owner anti-leak + dual caps.

no ad-hoc composition; any fail → BLOCK
Never stuck

Graceful fallback

Can’t compose, or a check fails? It degrades to the step-by-step chain — funds and every Guardian check unaffected. A SUI shortfall is reported plainly.
